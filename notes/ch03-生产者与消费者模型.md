## 生产者与消费者模型

上一章我们学习了线程的基础知识，知道了 `Thread.start()` 可以启动线程，`sleep()` 可以让线程休眠。本章我们来学习一个经典的多线程场景：**生产者与消费者问题**。

### 问题背景

想象一个咖啡店场景：
- **瑞幸（生产者）**：负责生产咖啡，但库存有限，最多放 3 杯
- **消费者**：有多个消费者，每人需要喝固定数量的咖啡

当库存满了，生产者要休息等待；当库存空了，消费者要等待生产者。

public class Producer extends Thread {
    private List<Coffee> coffees;
    private Object monitor;
    private int limit;

    public Producer(String name, int limit, List<Coffee> coffees) {
        super(name);
        this.limit = limit;
        this.coffees = coffees;
        this.monitor = coffees;
    }

    @Override
    public void run() {
        int id = 0;
        while (true) {
            synchronized (this.monitor) {
                if (coffees.size() >= this.limit) {
                    try {
                        System.out.printf("--%s 生产太多了，进入休息---%n", this.getName());
                        monitor.wait();
                        System.out.printf("---%s 恢复，继续生产咖啡---%n", this.getName());
                    } catch (InterruptedException e) {
                    }
                }

                Coffee coffee = new Coffee(++id, coffeeNames.get(id % coffeeNames.size()));
                coffees.add(coffee);
                System.out.printf("%s 生产%s %n", this.getName(), coffee.name());

        }
    }
}

[🧑 Consumer.java](https://github.com/upangka/ComicJava/blob/main/src/cn/comicjava/ch03/v1_notify_wait/Consumer.java)
```java
public class Consumer extends Thread {
    private List<Coffee> coffees;
    private int count;
    private Object monitor;

    public Consumer(String name, int count, List<Coffee> coffees) {
        super(name);
        this.count = count;
        this.coffees = coffees;
        this.monitor = coffees;
    }

    @Override
    public void run() {
        int drinked = 1;
        while (count >= drinked) {
            synchronized (monitor) {
                while (coffees.isEmpty()) {
                    try {
                        System.out.printf("店内繁忙，%s 等待中%n", this.getName());
                        monitor.wait();
                    } catch (InterruptedException e) {
                    }
                }

                Coffee coffee = this.coffees.remove(0);
                System.out.printf("%s 畅饮第(%d)杯 %s%n", this.getName(), drinked, coffee.name());
                drinked++;
                monitor.notifyAll();
            }
        }
        System.out.printf("%s 离店%n", this.getName());
    }
}
```

### 核心概念：synchronized + wait/notifyAll

#### 1. synchronized - 锁住共享资源

```java
synchronized (monitor) {
    // 操作共享变量 coffees
}
```

` synchronized` 保证同一时刻只有一个线程能进入这个代码块，避免数据混乱。

#### 2. wait() - 让线程等待

```java
// 生产者：库存满了，等待消费者来买
if (coffees.size() >= this.limit) {
    monitor.wait();  // 进入等待，释放锁
}

// 消费者：库存空了，等待生产者来补货
while (coffees.isEmpty()) {
    monitor.wait();
}
```

调用 `wait()` 后，线程会**进入等待状态并释放锁**，让其他线程可以进来。

#### 3. notifyAll() - 唤醒所有等待的线程

```java
// 取走咖啡后，通知生产者可以继续生产了
monitor.notifyAll();
```

调用 `notifyAll()` 会唤醒所有在这个对象上等待的线程，让它们重新竞争锁。

### 重要：while 与 synchronized 的位置顺序

这是最容易出错的地方！必须严格按照以下顺序：

#### ✅ 正确写法

```java
while (条件) {                    // 1. 用 while 循环检查条件
    synchronized (monitor) {      // 2. 在 synchronized 块内部调用 wait
        while (共享资源不满足) {   // 3. 再次用 while 检查条件
            monitor.wait();
        }
        // 处理共享资源
        monitor.notifyAll();      // 4. 操作完后通知其他线程
    }
}
```

#### ❌ 常见错误

**错误1：用 if 代替 while**
```java
// 错误！应该用 while
if (coffees.isEmpty()) {
    monitor.wait();  // 线程被唤醒后不会重新检查！
}
```

**错误2：wait 在 synchronized 外面**
```java
// 错误！会抛出 IllegalMonitorStateException
monitor.wait();  // 必须先获得锁
synchronized (monitor) {
    // ...
}
```

#### 为什么必须用 while？

因为存在 **虚假唤醒（Spurious Wakeup）** 的可能。Java 规范说 `wait()` 可能无缘无故返回，所以必须用 while 重新检查条件：

```java
while (coffees.isEmpty()) {  // 被唤醒后，重新检查是否真的有咖啡
    try {
        monitor.wait();
    } catch (InterruptedException e) {
    }
}
```

### 总结

| 概念 | 作用 |
|------|------|
| `synchronized` | 保证同一时刻只有一个线程操作共享资源 |
| `wait()` | 让线程等待，并释放锁 |
| `notifyAll()` | 唤醒所有等待的线程 |
| `while` | 循环检查条件，防止虚假唤醒 |

**口诀**：`while` 在 `synchronized` 里，`wait` 在 `while` 里，`notifyAll` 在最后。
